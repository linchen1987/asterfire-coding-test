笔试题目: AI 赋能的智能简历分析平台

## 背景
在招聘流程中，快速筛选和分析大量简历是一项耗时的工作。你的任务是构建一个 全栈应用，能够上传和解析简历（PDF 格式），利用 AI 模型提取关键信息并进行智能评分，同时提供一个 交互丰富、体验优秀 的前端界面来管理和可视化分析候选人数据。

## 技术要求
- frontend: nextjs
- backend: nestjs
- 数据存储: sqlite (单机)
- API 规范：RESTful
- 部署: 前端 vercel, 后端 render (https://render.com/)


## 功能模块

### 模块一：简历上传与解析
- 提供文件上传组件，支持 拖拽上传 和点击上传，限制 PDF 格式
- 支持 批量上传（至少同时上传 5 份简历），展示上传进度和状态
- 后端解析 PDF 内容，提取文本信息（需兼容多页简历）
- 对提取文本进行清洗和结构化处理
- 加分项：上传过程中展示实时预览（PDF 首页缩略图）

### 模块二：AI 智能信息提取
- 利用 AI 模型（如 OpenAI compatible api）从简历文本中提取以下结构化信息：
```
类别	字段	是否必须
基本信息	姓名、电话、邮箱、所在城市	✅ 必选
教育背景	学校、专业、学历、毕业时间	✅ 必选
工作经历	公司名称、职位、时间段、工作内容摘要	✅ 必选
技能标签	技术栈、工具、语言等（以标签形式提取）	✅ 必选
项目经历	项目名称、技术栈、个人职责、项目亮点	⭐ 加分项
```

- AI 提取过程通过 SSE（Server-Sent Events）流式返回，前端逐步渲染提取结果
- 加分项：支持用户在前端手动修正 AI 提取的信息

### 模块三：岗位匹配与智能评分
- 提供岗位需求配置界面（JD 编辑器），支持输入岗位描述、必备技能、加分技能
- 调用 AI 模型对简历与岗位需求进行匹配分析，返回： 
  - 综合匹配度评分（0-100 分）
  - 各维度子评分：技能匹配度、经验相关性、教育背景契合度
  - AI 评语：一段文字说明该候选人的优势与不足
- 评分结果以 可视化图表 展示（如雷达图 / 柱状图 / 环形进度条）
- 加分项：支持多个 JD 同时对比评分


### 模块四：候选人管理面板
- 实现候选人列表页，支持： 
  - 表格视图 / 卡片视图 切换
  - 按评分、上传时间、技能标签 排序与筛选
  - 关键字搜索（姓名、技能、学校等）
  - 分页
- 点击候选人进入 详情页，展示完整的解析信息、评分详情、原始 PDF 预览
- 支持候选人状态管理（待筛选 → 初筛通过 → 面试中 → 已录用 / 已淘汰），状态变更需有视觉反馈
- 加分项：支持候选人对比功能（选择 2-3 人，并排对比各维度评分）


## 质量要求

### 前端 ui 质量要求
- 页面整体需具备 良好的视觉设计和交互体验（不接受纯原生 HTML 样式）
- 实现 响应式布局，适配桌面端（≥1280px）
- 全局错误处理与 Loading 状态管理
- 加分项 
  - 暗色 / 亮色主题切换
  - 骨架屏（Skeleton）加载态
  - 键盘快捷键支持
  - 动画过渡效果

### 代码质量要求
- 使用 组件化架构，合理拆分组件粒度，体现复用性和可维护性
